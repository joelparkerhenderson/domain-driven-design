## Per-Domain Structure

Each domain includes:

### Phase 1: Scaffolding

- CLAUDE.md
- plan.md
- tasks.md
- ubiquitous-language.md
- index.md with README.md symlink
- topics/index.md with README.md symlink

### Phase 2: Strategic Design Topics (6)

- strategic-design, bounded-contexts, context-map, ubiquitous-language, subdomains, anti-corruption-layer

### Phase 3: Tactical Design Topics (6)

- aggregates, entities, value-objects, domain-events, repositories, domain-services

### Phase 4: Bounded Context Topics (5-6)

- Domain-specific bounded contexts

### Phase 5: Cross-Cutting Concern Topics (4)

- event-driven-architecture, integration-patterns, {domain}-standards, regulatory-compliance

### Phase 6: Finalize

- Verify all symlinks
- Cross-reference topics for consistency
- Update tasks.md

## Per-Topic Content Requirements

Each topic index.md must include:

- Title and overview
- Relevance to the specific domain
- Key concepts and terminology
- Examples within the domain context
- Relationships to other topics
- References and citations (books, whitepapers, articles)

Each topic directory must have:

- index.md - Comprehensive documentation
- README.md - Symlink to index.md
