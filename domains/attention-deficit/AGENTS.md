# Attention-Deficit Domain

Domain-Driven Design applied to ADHD management systems.

## Domain Overview

This domain models attention-deficit/hyperactivity disorder (ADHD) management
across clinical, educational, and therapeutic settings using DDD principles.

## Bounded Contexts

1. Clinical Assessment Context - diagnostic evaluation, rating scales, neuropsychological testing
2. Treatment Management Context - multimodal treatment plans, behavioral therapy, coaching
3. Behavioral Monitoring Context - symptom tracking, behavior charting, executive function monitoring
4. Educational Accommodation Context - IEP/504 plans, classroom strategies, academic support
5. Medication Management Context - stimulant/non-stimulant protocols, dosage optimization, side effects
6. Outcomes Tracking Context - treatment effectiveness, functional improvement, quality of life

## Ubiquitous Language

Use ADHD clinical and educational terminology consistently across all contexts.

## Permissions

- Do not read above this project directory.

## Guidelines

- Create comprehensive documentation using DDD strategic and tactical patterns.
- Do not create images, diagrams, web applications, front-end, or back-end code.
- Provide references and citations to DDD literature and ADHD clinical guidelines.
- Business logic must remain pure and independent of infrastructure concerns.
