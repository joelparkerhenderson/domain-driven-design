# Autism Domain

Domain-Driven Design applied to autism spectrum management systems.

## Domain Overview

This domain models autism spectrum disorder (ASD) management across clinical,
therapeutic, sensory, communicative, and educational settings using DDD principles.

## Bounded Contexts

1. Diagnostic Assessment Context - screening tools (M-CHAT, ADOS-2), diagnostic criteria (DSM-5), developmental evaluation
2. Behavioral Intervention Context - ABA therapy, behavior plans, skill acquisition, data collection
3. Sensory Processing Context - sensory profiles, sensory diet, environmental modifications
4. Communication Support Context - AAC devices, speech therapy, social communication, PECS
5. Educational Planning Context - IEP development, transition planning, inclusive education
6. Outcomes Tracking Context - developmental milestones, treatment effectiveness, quality of life

## Ubiquitous Language

Use autism clinical, therapeutic, and educational terminology consistently across all contexts.

## Permissions

- Do not read above this project directory.

## Guidelines

- Create comprehensive documentation using DDD strategic and tactical patterns.
- Do not create images, diagrams, web applications, front-end, or back-end code.
- Provide references and citations to DDD literature and autism clinical guidelines.
- Business logic must remain pure and independent of infrastructure concerns.
