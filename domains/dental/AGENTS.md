# Dental Domain - Domain-Driven Design

This domain applies Domain-Driven Design to dental practice systems,
covering clinical examination, treatment planning, restorative services,
orthodontic management, periodontal care, and practice management.

## Bounded Contexts

1. Clinical Examination Context
2. Treatment Planning Context
3. Restorative Services Context
4. Orthodontic Management Context
5. Periodontal Care Context
6. Practice Management Context

## Structure

- plan.md: Strategic plan for the dental domain.
- tasks.md: Task list with completion status.
- ubiquitous-language.md: Shared vocabulary for the dental domain.
- index.md: Overview and entry point for documentation.
- topics/: Directory of detailed topic documentation.

## Guidelines

- Use ubiquitous language consistently across all bounded contexts.
- Keep business logic pure and independent of infrastructure.
- Model dental workflows as domain events crossing context boundaries.
- Follow ADA standards and HIPAA regulatory compliance.
- Reference Evans 2003, Vernon 2013, Khononov 2021.
