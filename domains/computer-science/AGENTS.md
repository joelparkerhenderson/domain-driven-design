# Computer Science Domain - AGENTS.md

## Overview

This domain applies Domain-Driven Design (DDD) to computer science, encompassing algorithms and data structures, programming languages and compilers, operating systems and systems architecture, networking and distributed systems, databases and information management, and software engineering and design.

## Bounded Contexts

1. Algorithms and Data Structures - Manages the modeling of computational procedures for solving problems and the data organizations that support efficient access and modification, including complexity analysis and algorithmic paradigms.
2. Programming Languages and Compilers - Governs the design, semantics, and implementation of programming languages, including lexical analysis, parsing, type systems, code generation, and runtime environments.
3. Operating Systems and Systems Architecture - Handles the modeling of system software that manages hardware resources, process scheduling, memory management, file systems, and the architectural patterns of computing platforms.
4. Networking and Distributed Systems - Manages the protocols, architectures, and algorithms governing communication between computing nodes, including network layering, consensus protocols, fault tolerance, and distributed data management.
5. Databases and Information Management - Governs the theory and practice of data storage, retrieval, and management, including relational models, query languages, transaction processing, indexing, and data integrity.
6. Software Engineering and Design - Manages the systematic approaches to software development, including requirements engineering, architectural patterns, design principles, testing methodologies, and development lifecycle processes.

## Domain Principles

- Model using shared ubiquitous language between computer scientists, software engineers, systems architects, data engineers, and researchers.
- Group the system into bounded contexts reflecting the major subdisciplines of computer science.
- Business logic remains pure and independent of infrastructure (database, UI, web, mobile).
- Follow ACM Computing Curricula, IEEE Software Engineering standards, and established theoretical foundations.
- Maintain formal correctness, computational complexity awareness, and security as cross-cutting concerns.
